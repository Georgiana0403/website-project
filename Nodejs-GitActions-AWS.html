CI/CD Pipeline with GitHub Actions and AWS Elastic Beanstalk

Project Overview

Project Description

This project is a simple To-Do List application built with Node.js, Express.js, and HTML/CSS. The application is designed to be deployed on AWS Elastic Beanstalk, leveraging a CI/CD pipeline for seamless updates and maintenance.

Technologies Used

Node.js GitHub Actions CI/CD AWS Elastic Beanstalk

Key Features

  • Creation of a simple Node.js application
  • Support for multiple languages
  • Intuitive web interface
  • Machine learning model training

Project Steps

Clone the repository

To get started with the project locally, you have to clone the repository, install dependencies and start the application.

git clone https://github.com/Georgiana0403/nodejs-app.git

cd nodejs-app

npm install

npm start

Add AWS credentials to GitHub Secrets

Go to your GitHub repository settings and navigate to "Secrets and variables" > "Actions", after that add the following secrets:

AWS_ACCESS_KEY_ID

AWS_SECRET_ACCESS_KEY

Create an Elastic Beanstalk Application

Log in to the AWS Management Console, Navigate to Elastic Beanstalk and click on "Create a new application". Enter a name for your application, for this project the name is nodejs-app, and click "Create".

Create an Elastic Beanstalk Environment

After cloning the project, you need to create a zip version of that folder because it has to be uploaded when you configure the environment. After creating the application, click on "Create a new environment", select "Web server environment" and for platform choose "Node.js". For the environment name, you have to write the same name as it is in the script (nodejs-app-env). For "Application Code" choose "Upload your code" and upload the ZIP folder. At the "Instances" section, you have to choose as the "Root volume" the "General Purpose 3". After you finish all the configurations, click "Create environment".

Deployment

Ensure you have configured your AWS credentials and created the necessary environment before deploying. Now you can commit and push changes to the main branch. GitHub Actions will automatically build, test, and deploy the application to AWS Elastic Beanstalk.

git add .

git commit -m "Your commit message"

git push origin main
